  1. Lighthouse 评估工具

Lighthouse 评估内容 ​

一、性能方面 ​

  1. 得分：给出一个 0 到 100 之间的性能得分，直观地反映网页性能的优劣程度。

  2. 具体指标：

    • 首次内容绘制（FCP First Contentful Paint）时间：衡量用户看到页面上第一个内容的时间。
    • 最大内容绘制（LCP Largest Contentful Paint）时间：代表页面主要内容加载完成的时间。
    • 首次输入延迟（FID First Input Delay）：用户首次与页面交互时的响应时间。
    • 累积布局偏移（CLS Cumulative Layout Shift）：衡量页面在加载过程中布局的稳定性。
    • 总阻塞时间（TBT Total Blocking Time）：评估主线程被阻塞的时间总和。

  3. 问题分析：指出可能导致性能不佳的具体问题，如资源过大、过多的请求、未优化的图像等。

  4. 改进建议：提供针对性的优化建议，如压缩资源、减少请求数量、使用懒加载等。

二、可访问性方面 ​

  1. 得分：同样是 0 到 100 的可访问性得分。
  2. 问题清单：
    • 缺少图像的替代文本，影响视障用户通过屏幕阅读器理解内容。
    • 颜色对比度不足，可能对视力较弱的用户造成阅读困难。
    • 表单元素缺乏适当的标签或说明。
  3. 改进建议：针对发现的问题，给出具体的改进方法，如为图像添加准确的替代文本、调整颜色对比度、完善表单元素的标签等。

三、最佳实践方面 ​

  1. 得分：反映网页在最佳实践方面的表现。
  2. 分析内容：
    • 是否使用 HTTPS 协议确保安全连接。
    • 是否有效地利用缓存机制。
    • 是否正确设置了网页标题和描述。
    • 是否避免了常见的代码错误和不良实践。
  3. 改进方向：提出改进最佳实践的具体建议，如启用 HTTPS、优化缓存策略、完善网页元数据等。

四、SEO 方面 ​

  1. 得分：评估网页对搜索引擎优化的程度。
  2. 关键指标：
    • 页面标题的质量和相关性。
    • 元描述的优化程度。
    • 关键词的使用情况。
    • 是否有结构化数据以帮助搜索引擎理解页面内容。
  3. 优化建议：包括优化页面标题和描述、合理使用关键词、添加结构化数据等。

五、PWA 方面 ​

  1. 得分：衡量网页作为渐进式网络应用的性能。
  2. 特性检查：
    • 是否支持离线访问。
    • 是否可以添加到主屏幕。
    • 服务 worker 的使用情况。
  3. 发展建议：提供将网页打造成更好的 PWA 的建议，如实现离线功能、优化添加到主屏幕的体验、使用服务 worker 提高性能等。
